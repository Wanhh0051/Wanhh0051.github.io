# 项目管理场景配置

## 项目管理全流程

### 阶段1: 项目启动
```yaml
主导Agent: project-management-master
协作Agent:
  - goal-management-coach (目标设定)
  - time-efficiency-coach (时间规划)
质控Agent: code-quality-guardian
输出物:
  - 项目章程
  - 目标分解结构
  - 时间管理计划
  - 质量标准
```

### 阶段2: 规划执行
```yaml
主导Agent: project-management-master
协作Agent:
  - time-efficiency-coach (进度优化)
  - goal-management-coach (目标跟踪)
质控Agent: code-quality-guardian
输出物:
  - 详细执行计划
  - 资源分配方案
  - 风险应对策略
  - 沟通管理计划
```

### 阶段3: 监控控制
```yaml
主导Agent: time-efficiency-coach
协作Agent:
  - project-management-master (进度监控)
  - goal-management-coach (目标调整)
质控Agent: code-quality-guardian
输出物:
  - 进度监控报告
  - 偏差分析
  - 纠正措施
  - 预警通知
```

### 阶段4: 收尾总结
```yaml
主导Agent: project-management-master
协作Agent:
  - goal-management-coach (成果评估)
  - time-efficiency-coach (效率分析)
质控Agent: code-quality-guardian
输出物:
  - 项目总结报告
  - 经验教训文档
  - 最佳实践提炼
  - 改进建议
```

## 项目类型处理

### 敏捷项目管理
```yaml
Agent组合: project-management-master + time-efficiency-coach
处理流程:
  1. 敏捷转型规划
  2. 迭代计划制定
  3. 每日站会和回顾
  4. 冲刺管理和发布
  5. 持续改进和优化
```

### 多团队协作
```yaml
Agent组合: project-management-master + goal-management-coach
处理流程:
  1. 团队结构设计
  2. 协作机制建立
  3. 沟通渠道建设
  4. 冲突解决和协调
  5. 绩效评估和激励
```

### 风险管理
```yaml
Agent组合: project-management-master + time-efficiency-coach
处理流程:
  1. 风险识别和评估
  2. 风险应对策略制定
  3. 风险监控和预警
  4. 应急响应和恢复
  5. 风险总结和预防
```

### 资源优化
```yaml
Agent组合: time-efficiency-coach + project-management-master
处理流程:
  1. 资源需求分析
  2. 资源配置优化
  3. 利用率监控
  4. 调度和再平衡
  5. 效果评估和改进
```
